# 인형뽑기 (Claw Machine) — 스펙 (SDD)

바이브 코딩의 기준 문서. 각 요구사항은 ID를 가지며, **Phase 완료 = 해당 수용 기준(AC) 전부 통과**.
검증 방식: `[T]` = vitest 단위 테스트, `[C]` = 브라우저 플레이스루 체크리스트.

- [x] **R2.3** `[T][C]` 하강 버튼(Space/X, 화면 우측 버튼)을 눌러야만 하강이 시작되며, 최저점에서 손가락을 닫고 상승→원위치 복귀→개방까지 자동 진행

"다시 도전하기" 버튼을 누르면 점수·시도횟수가 0으로 리셋되고 인형이 랜덤으로 재채워진다(최고기록·크레딧은 유지). 배경 클릭 시 리셋 없이 닫힘 — `store.ts`(`showCelebration`/`roundsPlayed`/`dismissCelebration`/`retryChallenge`) + `Celebration.tsx` 구현, store 조작으로 승리→소진→오버레이 표시, 다시 도전하기(리셋 확인), 배경 클릭(리셋 없이 닫힘), 운영자 비우기(오버레이 미표시) 전부 브라우저에서 확인

## 아키텍처 규칙 (구현 시 준수)

1. **`src/logic/`은 Three/Rapier 임포트 금지** — 순수 함수만, 전부 테스트 대상
2. **매직넘버는 전부 `src/config.ts`** — 튜닝 세션은 단일 파일 수정
3. 클로는 `kinematicPosition` 바디, 이동은 반드시 `setNextKinematicTranslation()` (속도 클램프)
4. 컨테인먼트는 두꺼운(≥0.15) 투명 CuboidCollider — 얇은/트라이메시 콜라이더 금지
5. 타이머는 timestamp 기반 (`setTimeout` 금지)
6. 게임 상태: `IDLE → MOVING → DROPPING → GRABBING → LIFTING → RETURNING → RELEASING → RESULT → IDLE` (2026-07-09: MOVING_X/MOVING_Z 통합)
7. 라이선스: 의존성·에셋은 MIT/Apache-2.0/CC0만
